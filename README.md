# СРС №4: Темная/светлая тема с localStorage

Автор: Талгатулы Диас, ИИ-402

Репозиторий проекта: https://github.com/Fomar07/WebSRS4

Проект посвящен переключению светлой и темной темы. В работе объединены CSS-переменные, JavaScript-логика и Web Storage, а также показано, как переносить одно решение на несколько страниц.

## Что реализовано

- переключатель светлой и темной темы через JavaScript
- переопределение CSS-переменных для двух тем
- сохранение выбранной темы в `localStorage`
- восстановление темы при повторной загрузке страницы
- системная тема по умолчанию через `prefers-color-scheme`
- общая логика темы для нескольких страниц проекта
- синхронизация темы между вкладками через событие `storage`
- интерактивный квест в формате викторины на отдельной странице

## Структура проекта

- `index.html` — главная страница проекта
- `guide.html` — страница с архитектурой и масштабированием
- `quiz.html` — интерактивный квест по теме проекта
- `style.css` — общие стили и CSS-переменные
- `script.js` — логика темы и синхронизации между вкладками
- `quiz.js` — логика интерактивной викторины

## Соответствие требованиям

### Минимальный функционал

- есть HTML-страница с текстом, карточками, кнопками и переключателем темы
- реализованы светлая и темная темы

### Основной функционал

- тема переключается через JavaScript и CSS-переменные
- выбор пользователя сохраняется в `localStorage`
- при повторной загрузке страницы тема восстанавливается

### Расширенный функционал

- тема применяется на нескольких страницах за счет общего `script.js`
- тема синхронизируется между вкладками через `storage event`
- системная тема используется как значение по умолчанию
- в проекте показана разница между `localStorage` и серверным хранением

## Как запустить

1. Открыть `index.html` в браузере.
2. Переключить тему кнопкой.
3. Обновить страницу и проверить сохранение выбранной темы.
4. Перейти на `guide.html` и убедиться, что тема применяется там же.
5. Открыть `quiz.html` и пройти интерактивный квест.
6. Открыть проект в двух вкладках и проверить синхронизацию темы.

## Как работает проект

### CSS-переменные

В корне документа заданы переменные интерфейса: фон, цвет текста, акцент, тени и цвет поверхности. При смене темы меняется набор этих переменных, поэтому компоненты автоматически подстраиваются под новую тему.

### localStorage

При нажатии на кнопку текущая тема записывается в `localStorage` через `setItem`. При загрузке страницы проект читает значение через `getItem` и применяет сохраненную тему.

### Системная тема

Если тема еще не была выбрана вручную, проект использует результат `window.matchMedia('(prefers-color-scheme: dark)')` и подстраивается под системные настройки устройства.

### Синхронизация вкладок

Если тема меняется в одной вкладке, другие вкладки получают событие `storage` и обновляют интерфейс без перезагрузки.

## localStorage и сервер

### localStorage

- хранит данные только в браузере
- не требует базы данных и авторизации
- подходит для персональных настроек интерфейса
- не переносит настройки между устройствами автоматически

### Серверное хранение

- хранит настройки в базе данных или профиле пользователя
- позволяет получать те же настройки на разных устройствах после входа в аккаунт
- требует серверной логики, API и авторизации
- подходит для больших приложений с пользовательскими профилями

- синхронизация между вкладками
