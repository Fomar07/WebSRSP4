# Kahoot-квест: «Vue Quest Lab»

## Концепция

Квест строится как маршрут начинающего Vue-разработчика. Игрок проходит четыре сюжетные зоны:

1. Установка зависимостей.
2. Создание компонента.
3. Работа с реактивностью.
4. Передача props и финальная интеграция.

Рекомендуемый режим в Kahoot: обычная викторина с таймингом 20–30 секунд на вопрос.  
Количество вопросов: 15.

## Сценарий ведущего

Вступление:

«Вы попали в лабораторию Vue Quest Lab. Чтобы запустить приложение и дойти до релиза, нужно пройти 4 этапа: Setup, Components, Reactivity, Props. Каждый правильный ответ открывает следующий участок маршрута».

## Вопросы

### 1. Ворота запуска

- Сюжет: герой только создал пустую папку проекта.
- Вопрос: Какой инструмент сейчас рекомендуется для старта нового Vue-проекта?
- Варианты:
  - `create-vue` через Vite
  - Только Vue CLI
  - Только Webpack вручную
  - Только jQuery
- Правильный ответ: `create-vue` через Vite
- Тайминг: 20 сек
- Пояснение: Vue CLI находится в maintenance mode; для новых проектов рекомендуют `npm create vue@latest`.

### 2. Команда старта

- Сюжет: проект ещё не создан, нужна стартовая команда.
- Вопрос: Какая команда запускает официальный scaffolding Vue-проекта?
- Варианты:
  - `npm create vue@latest`
  - `vue start app`
  - `npm install vue-app`
  - `vite init vue-cli`
- Правильный ответ: `npm create vue@latest`
- Тайминг: 20 сек
- Пояснение: Именно эта команда указана в актуальном Tooling Guide Vue.

### 3. Архитектурный чертёж

- Сюжет: игрок открыл первый `.vue` файл.
- Вопрос: Из каких трёх основных блоков чаще всего состоит файл Single File Component?

  - `template`, `script`, `style`

  - `render`, `router`, `store`
  - `class`, `method`, `build`
- Правильный ответ: `template`, `script`, `style`
- Тайминг: 20 сек
- Пояснение: Это базовая структура `.vue` файла.

### 4. Разделение ответственности

- Сюжет: интерфейс начинает расти, один файл уже неудобен.
- Вопрос: Зачем приложение делят на компоненты?

  - Чтобы не использовать JavaScript
- Правильный ответ: Чтобы переиспользовать и упрощать код
- Тайминг: 20 сек
- Пояснение: Компоненты повышают читаемость и управляемость интерфейса.

### 5. Вход в реактивность

- Сюжет: игрок нашёл переменную `activeStageId` в `App.vue`.

- Правильный ответ: `ref`
- Тайминг: 20 сек
- Пояснение: `ref` оборачивает значение в реактивный объект.

### 6. Реактивный объект

- Сюжет: теперь герой работает с объектом `player`.
- Вопрос: Какой API чаще используют для реактивного объекта с несколькими полями?

  - `reactive`

  - `style`
- Правильный ответ: `reactive`
- Тайминг: 20 сек
- Пояснение: `reactive()` делает объект реактивным целиком.

### 7. Автоматический пересчёт

- Сюжет: на панели миссии выводится процент выполнения.
- Вопрос: Что лучше использовать для значения, которое зависит от других реактивных данных?
- Варианты:

  - `switch`
- Правильный ответ: `computed`
- Тайминг: 20 сек
- Пояснение: `computed` удобно для производных значений, например процента прогресса.

### 8. След в журнале

- Сюжет: в приложении записывается, когда пользователь переключает этап.
- Вопрос: Какой инструмент Vue позволяет отследить изменение реактивного значения?
- Варианты:

  - `slot`
  - `style scoped`
- Правильный ответ: `watch`
- Тайминг: 20 сек
- Пояснение: `watch` реагирует на изменение состояния и запускает побочный код.
